# Web Research Best Practices

## Research Strategy

### 1. Query Formulation
- Use specific, targeted search queries
- Include version numbers for technical topics (e.g., "React 18 hooks")
- Add context keywords ("tutorial", "documentation", "example")
- Search for official sources first

### 2. Multi-Source Verification
- Cross-reference information from at least 2-3 sources
- Prioritize official documentation
- Check publication dates for relevance
- Look for author credibility

### 3. Information Synthesis
- Organize findings by topic
- Identify conflicting information
- Note areas of uncertainty
- Summarize key takeaways

## Available Research Tools

### Web Search (Exa MCP)
- General web search for any topic
- Returns clean, extracted content
- Use for: documentation, tutorials, news, general research

### Company Research (Exa MCP)
- Business and market information
- Use for: competitive analysis, company background

### Code Context (Exa MCP)
- Programming-specific search
- Searches GitHub, Stack Overflow
- Use for: code examples, library docs, debugging help

## Research Workflow

1. **Define Scope**: What specific information do you need?
2. **Initial Search**: Cast a wide net with broad queries
3. **Deep Dive**: Follow up on promising leads
4. **Synthesize**: Combine findings into coherent summary
5. **Cite Sources**: Always reference where information came from

## Quality Indicators

✅ **Good Sources:**
- Official documentation
- Recent Stack Overflow answers (check date)
- GitHub repositories with active maintenance
- Academic papers and technical blogs
- News from reputable outlets

⚠️ **Use With Caution:**
- Outdated tutorials (check dates)
- Unverified forum posts
- Blog posts without citations
- AI-generated content without verification
